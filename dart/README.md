# fiducia_sync

Flutter local-first sync with durable SQLite storage and Supabase/backend
transports. The wire shapes match the canonical sync contracts generated by
`fiducia-interfaces`.

```dart
final store = await SqliteSyncStore.open('fiducia.db');
final sync = FiduciaSyncClient(store: store);
```

`SqliteSyncStore` atomically persists optimistic rows with their queue entries,
acknowledgements with queue retirement, server-wins conflict resolution, retry
counts, and incremental pull cursors.

For Supabase, create `SupabaseSyncTransport` with tenant-scoped realtime filters.
Its `pull()` uses the migration's `fiducia_sync_pull` RPC. Its `send()` requires
an application-owned write RPC; the package intentionally does not expose a
generic arbitrary-table mutation.

For the dependency-free Dart `fiducia_client`, adapt its JSON callbacks without
creating a package dependency in either direction:

```dart
final api = FiduciaClient('https://admin.fiducia.cloud');
final send = adaptJsonSender(
  api.syncSender(pathPrefix: '/api/admin/sync'),
);
final pull = adaptJsonPuller(
  api.syncPuller('infra_operations', pathPrefix: '/api/admin/sync'),
);

await sync.pull(pull);
await sync.optimisticWrite(
  table: 'infra_operations',
  id: operationId,
  row: {'id': operationId, 'state': 'queued'},
  send: send,
);
```

Run `flutter analyze` and `flutter test` from this directory.
